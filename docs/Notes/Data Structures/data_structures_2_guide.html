Implementing a Stack

Implement a stack data structure supporting push(), pop(), peek(), and isEmpty() operations.

Approach 1: Array-Based Stack

1
Implementation
class Stack: def __init__(self): self.items = [] def push(self, item): # Add to end of array - O(1) amortized self.items.append(item) def pop(self): if self.isEmpty(): raise Exception("Stack is empty") # Remove from end - O(1) return self.items.pop() def peek(self): if self.isEmpty(): raise Exception("Stack is empty") # View top without removing - O(1) return self.items[-1] def isEmpty(self): return len(self.items) == 0 def size(self): return len(self.items)
2
Complexity Analysis

Push: O(1) amortized

Pop: O(1)

Peek: O(1)

Space: O(n)

Array-Based Advantages

‚úì Simple implementation
‚úì Cache-friendly (contiguous memory)
‚úì No pointer overhead

Approach 2: Linked List Stack

1
Implementation
class Node: def __init__(self, data): self.data = data self.next = None class Stack: def __init__(self): self.top = None self._size = 0 def push(self, item): # Create new node and make it the new top new_node = Node(item) new_node.next = self.top self.top = new_node self._size += 1 def pop(self): if self.isEmpty(): raise Exception("Stack is empty") # Remove top and return its value data = self.top.data self.top = self.top.next self._size -= 1 return data def peek(self): if self.isEmpty(): raise Exception("Stack is empty") return self.top.data def isEmpty(self): return self.top is None def size(self): return self._size
2
Complexity Analysis

Push: O(1)

Pop: O(1)

Peek: O(1)

Space: O(n) + pointer overhead

Linked List Advantages

‚úì No need to resize
‚úì True O(1) push (no amortization)
‚úì Can grow indefinitely

Approach 3: Comparison

Array vs Linked List Stack

Aspect Array Linked List
Push/Pop O(1) amortized O(1) true
Memory Contiguous, cache-friendly Scattered, pointer overhead
Resizing Periodic O(n) cost No resizing needed
Implementation Simpler More complex
In practice, array-based stacks are usually faster due to better cache locality, despite occasional resize costs!

Valid Parentheses Problem

Given a string containing '(', ')', '{', '}', '[', ']', determine if the input string is valid. Valid means: brackets must close in correct order and every opening bracket has a corresponding closing bracket.

Approach 1: Stack-Based Solution

1
Implementation
def is_valid(s): # Map of closing to opening brackets matching = {')': '(', '}': '{', ']': '['} stack = [] for char in s: if char in matching: # Closing bracket if not stack or stack[-1] != matching[char]: return False stack.pop() else: # Opening bracket stack.append(char) # Valid if no unmatched opening brackets return len(stack) == 0 # Examples print(is_valid("()")) # True print(is_valid("()[]{}")) # True print(is_valid("(]")) # False print(is_valid("([)]")) # False print(is_valid("{[]}")) # True
2
How It Works

The algorithm uses a stack to track opening brackets:

  1. When we see an opening bracket, push it onto the stack
  2. When we see a closing bracket:
    • Check if stack is empty (no matching opening) ‚Üí invalid
    • Check if top of stack matches this closing bracket
    • If match, pop the opening bracket
    • If no match ‚Üí invalid
  3. At the end, stack should be empty (all brackets matched)
3
Complexity

Time: O(n) - one pass through string

Space: O(n) - worst case all opening brackets

Approach 2: Analysis & Walkthrough

1
Example: "{[]}"

Let's trace through a valid example:

Char Action Stack
{ Push ['{']
[ Push ['{', '[']
] Match & Pop ['{']
} Match & Pop []

Result: Stack empty ‚Üí Valid!

2
Example: "([)]" - Invalid
Char Action Stack
( Push ['(']
[ Push ['(', '[']
) ‚ùå Mismatch! Top is '[', not '('

Result: Mismatch detected ‚Üí Invalid!

Approach 3: Edge Cases

Important Edge Cases

Empty string "": Valid (no brackets to match)

Only opening "(((": Invalid (stack not empty at end)

Only closing ")))": Invalid (empty stack when trying to match)

Mismatched "([)]": Invalid (wrong bracket type at top)

Interleaved "({}[])": Valid (proper nesting)

This is a classic interview question! Practice until you can write it flawlessly in 5 minutes.
Always check if the stack is empty before popping or peeking to avoid errors!

Implementing a Queue

Implement a queue data structure supporting enqueue(), dequeue(), peek(), and isEmpty() operations.

Approach 1: Simple Array Queue

1
Naive Implementation
class Queue: def __init__(self): self.items = [] def enqueue(self, item): # Add to end - O(1) self.items.append(item) def dequeue(self): if self.isEmpty(): raise Exception("Queue is empty") # Remove from front - O(n) ‚ö†Ô∏è return self.items.pop(0) def peek(self): if self.isEmpty(): raise Exception("Queue is empty") return self.items[0] def isEmpty(self): return len(self.items) == 0
Dequeue is O(n) because removing from index 0 requires shifting all remaining elements!
2
Complexity

Enqueue: O(1)

Dequeue: O(n) ‚ö†Ô∏è INEFFICIENT

Approach 2: Linked List Queue

1
Optimal Implementation
class Node: def __init__(self, data): self.data = data self.next = None class Queue: def __init__(self): self.front = None # Remove from here self.rear = None # Add here self._size = 0 def enqueue(self, item): # Create new node new_node = Node(item) if self.rear is None: # Empty queue self.front = self.rear = new_node else: # Add to rear self.rear.next = new_node self.rear = new_node self._size += 1 def dequeue(self): if self.isEmpty(): raise Exception("Queue is empty") # Remove from front data = self.front.data self.front = self.front.next # If queue becomes empty if self.front is None: self.rear = None self._size -= 1 return data def peek(self): if self.isEmpty(): raise Exception("Queue is empty") return self.front.data def isEmpty(self): return self.front is None def size(self): return self._size
2
Complexity

Enqueue: O(1)

Dequeue: O(1)

Space: O(n)

Why Two Pointers?

Front pointer: For O(1) dequeue (remove from head)

Rear pointer: For O(1) enqueue (add to tail)

Without both, one operation would be O(n)!

Approach 3: Circular Buffer

1
Fixed-Size Circular Queue
class CircularQueue: def __init__(self, capacity): self.capacity = capacity self.queue = [None] * capacity self.front = 0 self.rear = -1 self._size = 0 def enqueue(self, item): if self.isFull(): raise Exception("Queue is full") # Wrap around using modulo self.rear = (self.rear + 1) % self.capacity self.queue[self.rear] = item self._size += 1 def dequeue(self): if self.isEmpty(): raise Exception("Queue is empty") item = self.queue[self.front] # Wrap around using modulo self.front = (self.front + 1) % self.capacity self._size -= 1 return item def isFull(self): return self._size == self.capacity def isEmpty(self): return self._size == 0
2
How Circular Buffer Works

Instead of shifting elements, we use modulo arithmetic to wrap around:

Initial:    [_, _, _, _]    front=0, rear=-1
Enqueue 1:  [1, _, _, _]    front=0, rear=0
Enqueue 2:  [1, 2, _, _]    front=0, rear=1
Dequeue:    [_, 2, _, _]    front=1, rear=1
Enqueue 3:  [_, 2, 3, _]    front=1, rear=2
Enqueue 4:  [_, 2, 3, 4]    front=1, rear=3
Enqueue 5:  [5, 2, 3, 4]    front=1, rear=0 (wrapped!)
3
Advantages

Enqueue: O(1)

Dequeue: O(1)

‚úì No shifting needed
‚úì Cache-friendly (array-based)
‚úì Fixed memory footprint

Circular buffers are perfect for fixed-size buffers in systems programming, like ring buffers in operating systems!
Summary: Stacks (LIFO) are perfect for function calls, undo/redo, and bracket matching. Queues (FIFO) excel at task scheduling, breadth-first search, and buffering. Master both!
